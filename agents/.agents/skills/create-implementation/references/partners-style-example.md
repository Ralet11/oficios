# Partners-Style Example

Use this file as a compact style anchor, not as fixed content to copy blindly.

## Signature structure

The reference style has these traits:

- start with `Source Scope`
- turn the raw source into a numbered list of implementation points
- follow with `Operational Delivery Blocks`
- add short notes for already-completed or sequencing-sensitive work
- create a dedicated frontend plan when UI is a major workstream

## Condensed outline

```md
# <Feature> Implementation

## Source Scope
- source file A
- source file B

## Implementation Tasks Ordered by Importance
### 1. Core task
### 2. Another task

## Operational Delivery Blocks
### Block 1. Foundation
Status:
- proposed
Tasks included:
- `1`
Scope:
- shared contract work
Deliverable:
- stable base

## Notes
- sequencing clarifications

## Frontend / UI Plan
### Scope
### Product Goals
### Visual Direction
### Route Map
### Core Screens
### Component Strategy
### Frontend Delivery Recommendation by Block
```

## Style cues

- keep headings operational
- make task titles short and specific
- use delivery blocks to explain efficient execution order
- keep the frontend or UI section detailed when the scope deserves it
- state when completed foundation work should not be replanned
